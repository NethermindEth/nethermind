# RpcTests.Generator

Generates RPC test case files (compatible with the [Erigon rpc-tests](https://github.com/erigontech/rpc-tests/) format) from JSONL request recordings.
Main use case - reports generated by Nethermind via `--JsonRpc.RpcRecorderState`.

**Support two modes:**
- **Report mode** (for files obtained from `--JsonRpc.RpcRecorderState All`):
  responses are already in the file; the tool matches them to requests by ID.
- **Live mode** (for `--JsonRpc.RpcRecorderState Request`):
  requests-only file; the tool sends each request to a reference client and records the response.

## Options

| Option | Short | Default | Description                                                                                   |
|--------|-------|---------|-----------------------------------------------------------------------------------------------|
| `--requests` | `-r` | *(required)* | JSONL input file(s). Append `:N` to start from line N (e.g. `file.jsonl:50000`)               |
| `--client` | `-c` | — | Reference client URL. If omitted, responses must already be in the file                       |
| `--parallelism` | `-p` | `1` | Number of concurrent requests to the reference client                                         |
| `--methods` | `-m` | — | Include only requests with these method names, repeatable                                     |
| `--min-blocks` | | — | Min block range for `eth_getLogs` requests (`toBlock - fromBlock`)                            |
| `--max-blocks` | | — | Max block range for `eth_getLogs` requests                                                    |
| `--min-result-len` | | — | Min number of entries in the response `result` array                                          |
| `--out` | `-o` | `{FileDir}/{FileName}.test.json` | Output path template (SmartFormat, see [TestCase.cs](./TestCase.cs) for available properties) |

## Examples

Generate tests from a full recording (obtained via `--JsonRpc.RpcRecorderState All`):
```bash
dotnet run -- -r rpc.1.txt
```

Generate tests by sending requests to a reference node (for `--JsonRpc.RpcRecorderState Request`:
```bash
dotnet run -- -r rpc.2.txt -c http://localhost:8545 -p 4
```

Filter `eth_getLogs` requests, limiting range and result length:
```bash
dotnet run -- \
  -r rpc.1.txt rpc.2.txt \
  -m eth_getLogs \
  --min-blocks 10 --max-blocks 1000 \
  --min-result-len 1
```

Custom output path per request (since Erigon test runner expects a single file per test)
```bash
dotnet run -- -r rpc.1.txt -o "{FileDir}/out/test_{TestN:0000}.json"
```

## Output format

```json
[
  {
    "request": {
      "jsonrpc": "2.0",
      "id": 1,
      "method": "eth_blockNumber",
      "params": []
    },
    "response": {
      "jsonrpc": "2.0",
      "id": 1,
      "result": "0x12345"
    }
  }
]
```
